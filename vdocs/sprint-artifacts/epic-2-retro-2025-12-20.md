## Challenges & Growth Areas

### 1. Performance Issues Identified
- **Project Card Opening Lag:** 1.5-2.5 second delay, especially for scanned projects
- **Root Cause Unknown:** Could be hardware-specific or actual performance bottleneck
- **Impact:** Affects user experience, particularly with large project collections
- **Investigation Needed:** Profiling required to identify bottleneck source

### 2. User Interface Regressions
- **AppShell Position Bug:** Window controls disappear when scrolling with many projects
- **Critical UX Issue:** Users lose access to minimize, close, settings buttons
- **Root Cause:** AppShell position not absolute, affected by page scroll
- **Impact:** Fundamental desktop application functionality compromised

### 3. Project Status Calculation Regression
- **Recurring Issue:** Git repositories showing Active status when dormant >1 week
- **Fixed Twice Previously:** Suggests underlying logic instability
- **Quality Concern:** Regression indicates insufficient test coverage for edge cases
- **User Impact:** Misleading project health information affects decision making

### 4. Theme and Visual Polish Issues
- **Dark Mode Placement:** Currently in AppShell, should be in Settings following standard UX patterns
- **Transition Quality:** Mode switching feels jarky with sudden color changes
- **Future Scalability:** Current placement doesn't accommodate multiple themes or customization

### 5. Gemini Story Creation Inconsistency
- **Variable Quality:** Sometimes thorough project context scanning, sometimes superficial
- **Fantasy Implementations:** Occasionally references non-existent file paths or components
- **Impact on Development:** Inconsistent story quality affects dev agent efficiency and accuracy
- **Root Cause Identified:** create-story workflow lacks mandatory codebase verification
